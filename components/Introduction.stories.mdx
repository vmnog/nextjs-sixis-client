# Bem-vindo ao Sixis Storybook

Utilizamos o Storybook para facilitar a manutenção e testes dos componentes de forma isolada.

Você pode observar que todos os componentes da pasta `./components/` tem um arquivo `index.js`, `styles.js` e um `.storie.js`.

Navegue pelas pastas e arquivos do projeto para se familiarizar e siga o padrão de criar os componentes isolados antes de implementa-los no restante do projeto em [**sixis-client**](https://github.com/vmnog/sixis-client).
